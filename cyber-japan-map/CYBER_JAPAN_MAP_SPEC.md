# CyberJapanMap 開発仕様書

## 1. プロジェクト概要
日本全国47都道府県の制覇状況を可視化・記録するWebアプリケーション。
サイバーパンク風のデザインを採用し、訪問記録（日付、メモ、評価、写真）をブラウザ内に保存します。

## 2. 技術スタック
- **フレームワーク**: Next.js 15 (App Router)
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS v4
- **データベース**: IndexedDB (Dexie.js) - ブラウザ内ローカルDB
- **地図ライブラリ**: @react-map/japan (SVGベース)
- **アイコン**: Lucide React
- **アニメーション**: Framer Motion

## 3. ディレクトリ構造と役割
```
src/
├── app/
│   ├── globals.css      # 全体のデザイン定義（サイバーパンク色の変数など）
│   ├── layout.tsx       # アプリの共通レイアウト
│   └── page.tsx         # メイン画面（地図、プログレスバーの表示）
├── components/
│   ├── JapanMap.tsx        # 日本地図の描画・色塗りロジック
│   └── PrefectureModal.tsx # 記録入力用のポップアップ画面
├── data/
│   └── prefectures.ts   # 都道府県のID・名前・コードの定義データ
├── db/
│   └── index.ts         # データベースの初期化・テーブル定義
└── hooks/
    └── useVisits.ts     # データの読み書きを行うための便利機能（カスタムフック）
```

## 4. データ構造 (Database Schema)
ブラウザのIndexedDBに `visits` というテーブルを作成して保存しています。

| 項目名 | 型 | 説明 |
| --- | --- | --- |
| id | number | 自動採番されるID (主キー) |
| prefectureId | number | 都道府県ID (1~47) |
| date | Date | 訪問日 |
| memo | string | メモ・感想 |
| rating | number | 5段階評価 |
| photo | Blob | 写真データ（画像ファイルそのもの） |
| createdAt | Date | 作成日時 |
| updatedAt | Date | 更新日時 |

## 5. 実装プロセス（開発順序）

### Step 1: 環境構築とデザイン基盤
- Next.jsプロジェクトを作成。
- `globals.css` にサイバーパンク用の色（ネオンシアン、マゼンタ、黒）とグリッド背景を定義。

### Step 2: データベース層の実装 (`src/db/`, `src/hooks/`)
- アプリの土台となる「データの保存場所」を先に作成。
- `Dexie.js` を使い、ブラウザにデータを保存する仕組みを構築。
- `useVisits` フックを作り、コンポーネントから簡単に「保存」「取得」「削除」ができるようにした。

### Step 3: データ定義 (`src/data/`)
- 地図とDBをつなぐための「都道府県データ（IDと名前の対応表）」を作成。
- 北海道などの特殊な文字コードに対応するためのマッピングもここで行う。

### Step 4: 地図コンポーネントの実装 (`src/components/JapanMap.tsx`)
- ライブラリを使って日本地図を表示。
- 保存されたデータ (`visits`) を読み込み、訪問済みの場所をピンク色 (`#ff00ff`) に塗るロジックを追加。

### Step 5: 入力モーダルの実装 (`src/components/PrefectureModal.tsx`)
- 地図をクリックした時に出る入力画面を作成。
- 写真のアップロード機能や、5段階評価のUIを実装。
- Framer Motionを使って、ふわっと表示されるアニメーションを追加。

### Step 6: メイン画面への統合 (`src/app/page.tsx`)
- 全ての部品を組み立て。
- 全国制覇率（プログレスバー）の計算ロジックを追加。
- ヘッダーやフッターを配置し、全体のレイアウトを整えた。

# 🔰 CyberJapanMap 徹底解説ガイド

ここからは、上記の仕様書の内容を、プログラミング初心者の方にも分かりやすく、用語解説やたとえ話を交えて解説します。

## 💡 そもそも、なぜ無料でこんなアプリが作れるの？

このアプリが無料で動いているのには、主に3つの理由があります。

1.  **サーバーを使っていないから (Serverless-ish)**
    *   通常のWebアプリは、データを保存するために月額料金がかかる「サーバー（巨大なコンピュータ）」を借ります。
    *   しかし今回は、**あなたのブラウザ（ChromeやEdge）の中にある保存領域（IndexedDB）** を使っています。
    *   **たとえるなら**: レンタル倉庫（有料サーバー）を借りずに、自宅の空き部屋（自分のブラウザ）に荷物を置いているようなものです。だから家賃（サーバー代）がかかりません。

2.  **オープンソースを使っているから (Open Source)**
    *   Next.js、React、Tailwind CSS、地図データなど、世界中のエンジニアが「自由に使っていいよ」と公開している無料の道具（ライブラリ）を組み合わせて作っています。
    *   **たとえるなら**: 自分でレンガを焼くところから家を作るのではなく、無料で配られている高品質なレゴブロックを組み立てて家を作ったようなものです。

3.  **あなたのパソコンで動かしているから (Localhost)**
    *   今はまだインターネット上に公開しておらず、あなたのパソコンの中で動いています。これを世界中に公開しようとすると、Vercelなどの無料プランを使うか、有料サーバーが必要になります。

## 🏗️ 用語解説と実装ステップの深掘り

### 1. 技術スタックって何？（道具箱の中身）

料理に例えると、以下のような道具を使っています。

*   **Next.js (ネクストジェイエス)**: **「システムキッチン」**
    *   Reactというライブラリを使いやすくまとめたフレームワークです。Webアプリを作るための便利な機能（ルーティングなど）が最初から揃っています。
*   **TypeScript (タイプスクリプト)**: **「設計図付きの食材」**
    *   JavaScriptに「型（Type）」というルールを加えたものです。「ここは数字しか入れちゃダメ」「ここは文字だけ」と決めることで、料理の失敗（バグ）を防ぎます。
*   **Tailwind CSS (テイルウィンドシーエスエス)**: **「味付け済みの調味料セット」**
    *   「文字を赤くする」「右に寄せる」といったデザインを、クラス名を書くだけで簡単に適用できるツールです。CSSを１から書く手間が省けます。
*   **IndexedDB (インデックスドディービー)**: **「ブラウザの中の金庫」**
    *   ブラウザの中に標準で備わっているデータベースです。今回はここに訪問記録を保存しています。

### 2. ディレクトリ構造って何？（家の間取り）

プログラムのファイルは、役割ごとに整理整頓してフォルダに入れています。

*   `src/app/`: **「リビング・玄関（画面）」**
    *   ユーザーが実際に見るページや、全体のレイアウト（壁紙や家具の配置）が入っています。
*   `src/components/`: **「家電・家具（部品）」**
    *   「地図」や「入力画面」など、特定の機能を持った部品が入っています。これらをリビング（app）に配置して画面を作ります。
*   `src/db/` & `src/hooks/`: **「配管・電気配線（裏側の仕組み）」**
    *   データの保存や読み出しを行う、目に見えない重要な仕組みが入っています。

### 3. 実装プロセス詳細解説

#### Step 1: 環境構築とデザイン基盤
*   **何をした？**: アプリの土台を作り、世界観（サイバーパンク）を決めました。
*   **たとえるなら**: 家を建てる土地を整地し、「今回は近未来風の黒い家にしよう！」と壁紙や照明（ネオンカラー）を用意した段階です。

#### Step 2: データベース層の実装
*   **何をした？**: データを保存する「箱」を用意しました。
*   **たとえるなら**: 家の中に「思い出ボックス」を用意し、「ここには日付と写真と感想を入れる」というルール（テーブル定義）を決めた段階です。
*   **ポイント**: `Dexie.js` という道具を使うことで、難しいIndexedDBの操作を簡単にしています。

#### Step 3: データ定義
*   **何をした？**: 地図上の「北海道」と、データの「北海道」を結びつける名札を作りました。
*   **たとえるなら**: 地図のピース一つ一つに「これはID番号1番の北海道ですよ」とタグ付けをして、システムが理解できるようにしました。

#### Step 4: 地図コンポーネントの実装
*   **何をした？**: 日本地図を表示し、行った場所を塗る仕組みを作りました。
*   **たとえるなら**: 白地図を用意し、「思い出ボックスに記録がある県は、蛍光ペン（ピンク）で塗りつぶす」というルールをプログラムしました。

#### Step 5: 入力モーダルの実装
*   **何をした？**: クリックした時に出る「詳細入力画面」を作りました。
*   **たとえるなら**: 地図の県をクリックすると、「いつ行った？」「楽しかった？」と聞いてくるアンケート用紙（フォーム）が飛び出してくるようにしました。写真もクリップで留められます。

#### Step 6: メイン画面への統合
*   **何をした？**: 全ての部品を合体させ、完成させました。
*   **たとえるなら**: 用意した「地図」「アンケート用紙」「思い出ボックス」をリビングに配置し、さらに壁に「全国制覇まであと◯％」というグラフ（プログレスバー）を貼り付けました。
