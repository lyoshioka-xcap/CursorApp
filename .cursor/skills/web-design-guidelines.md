# Web Design Guidelines

このスキルは、WebアプリケーションのUI/UXデザインに関するベストプラクティスを提供します。

## 1. レスポンシブデザイン (Responsive Design)
- **モバイルファースト**: スマートフォン向けのレイアウトを優先して設計し、PC向けに拡張する。
- **ブレークポイント**: 一般的なブレークポイント（例: 640px, 768px, 1024px, 1280px）を使用する。
- **フレキシブルレイアウト**: 固定幅（px）ではなく、相対幅（%, rem, vw）を使用して柔軟に対応する。

## 2. アクセシビリティ (Accessibility)
- **カラーコントラスト**: テキストと背景のコントラスト比を確保する（WCAG基準）。
- **セマンティックHTML**: `<div>` だけでなく `<header>`, `<main>`, `<button>` などの意味のあるタグを使用する。
- **キーボード操作**: マウスを使わずにTabキーだけで操作できるようにする。
- **代替テキスト**: 画像には必ず `alt` 属性を設定する。

## 3. タイポグラフィ (Typography)
- **フォントサイズ**: 本文のベースサイズは16px（1rem）を推奨。
- **行間 (Line Height)**: 本文は1.5〜1.7、見出しは1.2〜1.3を目安にする。
- **階層構造**: 見出し（h1, h2...）を使って情報の優先順位を明確にする。

## 4. スペーシングとレイアウト (Spacing & Layout)
- **8pxルール**: マージンやパディングは8の倍数（8px, 16px, 24px...）で統一すると整いやすい。
- **ホワイトスペース**: 余白を恐れずに使い、要素ごとの関連性を視覚的に表現する。
- **グリッドシステム**: 12カラムグリッドなどを活用して配置を整列させる。

## 5. UIコンポーネント (UI Components)
- **ボタン**: クリック可能な領域を十分に確保する（タッチデバイスでは44px以上推奨）。
- **フォーム**: 入力欄には明確なラベルを付け、エラーメッセージは色だけでなくテキストでも伝える。
- **フィードバック**: ホバー時やクリック時に視覚的なフィードバック（色変化など）を与える。

## 6. パフォーマンス (Performance)
- **画像の最適化**: WebP形式の使用や遅延読み込み（Lazy Loading）を検討する。
- **アニメーション**: スクロールや操作を阻害しない、スムーズなアニメーション（CSS transform/opacity）を使用する。
