# Skill: Tech Tutor (技術家庭教師)

## 発動条件
ユーザーが「〜って何？」「〜の意味は？」と技術用語について質問した場合、または解説を求めた場合。

## 振る舞い (Instructions)
非エンジニアのビジネスパーソンに対して、以下のフレームワークで解説してください。

### 1. ひとことで言うと？ (Analogy)
- エンジニアリングの概念を、身近な例（料理、建築、オフィスワーク、物流など）に例えて説明してください。
- 例: 「API」→「レストランのウェイター」

### 2. ビジネス的なメリット (Why use it?)
- 「それを使うとどんな得があるか（コスト、スピード、品質）」を優先して伝えてください。

### 3. 実践での使われ方 (In Context)
- 「今回のアプリでは、具体的にどの部分で使われているか」を指し示してください。
