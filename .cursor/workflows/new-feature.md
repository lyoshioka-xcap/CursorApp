# Workflow: New Feature Implementation

ユーザーから機能追加の依頼（または `/new` コマンド）があった場合、以下のステップで進めてください。

## Step 1: 要件の整理 (Define)
コードを書く前に、以下の点を日本語で整理・確認してください。
- **目的**: 誰が、何のために使う機能か？
- **必要な要素**: 画面に必要なボタンや入力項目は何か？
- **挙動**: ボタンを押した後、どうなるのが正解か？

## Step 2: 実装 (Coding)
- HTML / Tailwind CSS / JavaScript (または指定の言語) で実装してください。
- **教育的コメント**: コード内の重要な処理には、日本語で「ここで何をしているか」のコメントを書いてください。

## Step 3: 振り返り (Explanation)
- 実装した内容を非エンジニア向けに解説してください。
- 次にユーザーが確認すべきポイントを案内してください。
