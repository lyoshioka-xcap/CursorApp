# Workflow: Code Review

ユーザーからコードの確認依頼（または `/review` コマンド）があった場合、以下の視点でチェックしてください。

## 1. バグと安全性 (Bugs & Security)
- エラーの原因になる記述はないか？
- 個人情報やAPIキーが露出していないか？

## 2. 可読性 (Readability)
- 変数名は分かりやすいか？（`x`, `data` などの曖昧な名前は避ける）
- 後から見返して理解できる構造になっているか？

## 3. 改善アドバイス (Advice)
- 「もっと良くするならこう書ける」というTipsを1つ提案してください（理由とともに）。
